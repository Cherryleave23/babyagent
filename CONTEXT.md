# 母婴垂类 B2B Agent

面向母婴企业门店员工的 AI 助手，充当"智能儿科医生 + 儿童营养师"角色，辅助为 0-6 岁宝宝定制膳食方案、推荐本公司产品、提供儿科健康咨询。

## 术语表（Language）

**宝宝（Baby / Child）**：
0-6 岁的婴幼儿，是 Agent 服务的唯一目标人群。
_避免_：儿童、幼儿、小孩

**宝宝档案（Baby Profile）**：
存储在端侧的宝宝个体数据记录，包含年龄、性别、过敏史、生长发育数据、历史咨询记录等。端侧本地保留，严禁幻觉。
_避免_：用户档案、客户档案

**端侧（Edge / Enterprise Side）**：
部署在企业/门店本地的 Agent 实例，拥有该企业的专属数据库（产品库、员工账号、宝宝档案）。
_避免_：本地、客户端

**总部大知识库（Central Knowledge Base）**：
由母婴知识运营团队维护的全局母婴知识库，所有端侧实例共享，不断扩展。
_避免_：云端知识库、总知识库

**基座模型（Base LLM）**：
Agent 调用的在线大语言模型，不绑定单一厂商，支持 DeepSeek、GLM、ChatGPT 等可替换。
_避免_：模型、AI 模型

**产品推荐（Product Recommendation）**：
基于宝宝档案和需求，从该企业产品库中推荐合适产品，严禁推荐企业产品库中不存在的产品（严禁幻觉）。

**门店（Store）**：
部署一个 Agent 实例的物理母婴门店，一个门店一个 Agent，多个员工共用。
_避免_：店铺、分店

**员工（Employee / Staff）**：
门店内通过微信与 Agent 交互的工作人员，对话间相互隔离。
_避免_：用户、客户

**备注（Note）**：
Agent 在对话中自动提取、写入宝宝档案的非结构化特征信息（如饮食偏好、新发现的过敏反应、特殊情况）。LLM 自主生成，推送通知 + 可撤销，不参与产品推荐决策。
_避免_：笔记、标记

**总部种子库（Seed Knowledge Base / 层A）**：
部署时预装、云端定时同步的通用母婴医学知识库（疾病、营养学、疫苗、膳食模板）。端侧只读，带版本号+时间戳。
_避免_：总部知识库、云端知识库

**企业拓展库（Enterprise Extension DB / 层B）**：
企业自录入的企业专属数据（产品SKU、自有配方、定制模板），通过 Web 控制台管理。
_避免_：企业数据库、产品库

**运行时数据（Runtime Data / 层C）**：
Agent 在运行中产生的数据（宝宝档案、对话历史、技能记忆），由 Agent 自主写入。
_避免_：动态数据

**端侧统一数据库（Unified Edge DB）**：
层A+层B+层C 合并为同一个向量检索空间，运行时无层标识，员工无感。
_避免_：端侧数据库

**宝宝指针（Active Baby Pointer）**：
每个员工会话中当前活跃的宝宝标识，支持隐式（从消息中识别宝宝名）和显式（员工指令）两种切换方式。
_避免_：当前宝宝、活跃宝宝

**Web 管理控制台（Admin Console）**：
部署在企业端的 Web 配置界面，供 IT 管理员进行微信绑定、API Key 配置、员工管理、产品录入等运维操作。不是聊天界面。
_避免_：后台、管理后台

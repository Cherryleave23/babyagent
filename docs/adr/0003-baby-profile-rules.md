# ADR-0003：宝宝档案指令驱动 + LLM 只读架构

**日期**：2026-07-19  
**状态**：已接受

## 背景

宝宝档案是 Agent 的核心数据结构，包含过敏史、饮食禁忌等关键医学信息。档案的准确性直接影响产品推荐和健康建议的质量。允许 LLM 自由写入档案存在以下风险：LLM 可能误解对话中家长随口提及的信息为确定事实、可能覆盖已确认的正确档案数据。

## 决策

**宝宝档案采用指令驱动（规则）写入 + LLM 只读架构**：

1. **核心字段**（姓名、性别、出生日期、过敏史、饮食禁忌、生长发育记录）——创建与修改走指令解析（`@Agent 建档/更新`），纯规则驱动，不经过 LLM 生成
2. **非核心字段（备注）**——LLM 自主提取，推送通知 + 可撤销，不参与产品推荐决策
3. **LLM 访问权限**——仅可读取档案作为对话上下文，不得直接修改核心字段
4. **关键字段保护**——过敏信息变更走 append-only 策略，LLM 不可发起删除

## 备选方案

**方案 B**：全字段 LLM 自由读写。
风险：幻觉导致的档案错误会连锁污染产品推荐和健康建议。

**方案 C**：全字段指令驱动（包括备注）。
缺陷：员工的非正式对话中的有用信息会被遗漏，降低业务效率。

## 后果

- **优点**：核心数据零幻觉，过敏信息不会丢失（append-only），变更可追溯（审计日志）
- **妥协**：备注字段允许 LLM 写入，但通过 `备注≠推荐输入` 的不变式隔离风险

# ADR-0004：上下文压缩双输出 —— Session 私有摘要 + 宝宝档案变更建议

**日期**：2026-07-19  
**状态**：已接受

## 背景

多员工共同服务同一宝宝，但每个员工的对话（session）严格隔离。会话隔离保证信息不外泄和责任清晰，但宝宝状态变化（如新发现的过敏信息）必须跨 session 共享。传统的上下文压缩只生成一份 session 私有摘要，不能同时满足隔离与共享的双重需求。

## 决策

**上下文压缩输出两份结构化内容**：

1. **Session 私有摘要**——仅本 session 可用，不跨 session 传播。包含员工对客户情绪的主观判断、内部备注型对话、临时承诺等。防止信息噪声与责任混淆。
2. **宝宝档案变更建议**——结构化指令（`{field, operation, value, confidence}`），跨 session 共享。由系统执行写入，LLM 不直接覆写档案。仅 `confidence: high` 的变更自动生效，低置信变更写入待确认队列。

**压缩执行顺序**：（1）先执行宝宝状态提取+更新，（2）再基于更新后的档案生成 session 私有摘要。

**档案并发写入**：更新时间戳 Last-Write-Wins + 关键字段 append-only + 审计日志（来源 session_id + 时间戳 + 变更内容）。

## 后果

- **优点**：session 隔离与宝宝状态共享同时满足；档案变更可追溯；关键信息（过敏）不会丢失
- **代价**：压缩 LLM 调用一次需输出两份内容，token 消耗略增；压缩模块从"纯摘要"升级为"摘要+档案维护"双职责
